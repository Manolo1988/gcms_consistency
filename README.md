## 一、创新点及其解决的实际问题

### 创新点 1：基于度量学习的开放式产品识别与一致性评估

**解决的实际问题：**
- **新产品上线成本高**：传统分类器（SVM、CNN softmax）是闭集模型，每增加一个新烟草品类，需要收集大量样本 → 重新标注 → 重新训练 → 重新验证，周期长达数周
- **高频产品迭代不匹配**：烟草企业经常推出新品或调整配方，质控系统无法跟上
- **样本稀缺问题**：新产品初期可能只有几批数据，传统方法无法建立稳定模型

**技术方案：** 不学习"12个类别的分类边界"，而是学习一个**通用的嵌入空间**，使同类产品在空间中聚拢、不同类产品远离。新产品只需 3~5 个参考样本计算原型即可纳入系统。

**实际效益：** 新产品从采样到上线质控从"数周训练"缩短到"数小时录入参考样本"

---

### 创新点 2：批次不变表征学习

**解决的实际问题：**
- **RT 漂移**：不同批次间色谱柱老化、环境温度等导致保留时间系统性偏移，同一物质在不同批次的 RT 位置不同
- **灵敏度漂移**：检测器响应随时间衰减，导致同一浓度在不同批次的强度不一致
- **假阳性/假阴性**：批次效应会被误判为"不一致"（假阳性），或掩盖真实的产品偏差（假阴性）
- **传统校正的局限**：手工 RT 对齐（如 CODA、correlation optimized warping）需要人工调参，且无法完全消除非线性漂移

**技术方案：** 通过对抗训练/批次对比学习，让嵌入表征只保留产品特征、丢弃批次特征。

**实际效益：** 跨批次质控无需手工校准，批次间比较结果可靠

---

### 创新点 3：RT × m/z 双轴结构化注意力

**解决的实际问题：**
- **信息冗余与关键信号稀疏**：RT × m/z 矩阵中大部分区域是背景噪声，关键峰只占极小比例。普通 CNN 对所有位置等权处理，效率低
- **RT 轴和 m/z 轴语义不同**：RT 轴反映化合物洗脱顺序（有时序性），m/z 轴反映碎片离子模式（有组合性）。标准 2D 卷积无法区分这两个维度的不同物理含义
- **全局指纹与局部峰的双重重要性**：质控既关注整体轮廓的一致性，也关注特定标志物峰的存在与否

**技术方案：** 在 RT 方向和 m/z 方向分别施加自注意力，再融合，使模型能分别建模色谱分离模式和质谱碎裂规律。

**实际效益：** 相比直接用 ResNet 等通用 CNN，特征提取更高效，在有限样本下精度更高

---

### 创新点 4：可量化、可解释的一致性评分

**解决的实际问题：**
- **二元判定信息不足**："合格/不合格"无法告诉操作员偏差程度和偏差方向，不利于追溯原因
- **人工感官评价主观性强**：传统烟草质评依赖评审员打分，评审员间一致性差（ICC 通常 < 0.7）
- **质控溯源成本高**：检出不合格后，需要额外测试才能定位是哪些化学成分异常

**技术方案：** 基于嵌入距离输出连续一致性分数 [0,1]，并用 Grad-CAM 将偏差溯源到具体的 RT × m/z 区域 → 对应具体化合物。

**实际效益：** 一次推理同时完成"产品识别 + 偏差量化 + 异常溯源"三合一

## 二、技术框架设计

```
┌─────────────────────────────────────────────────────────────────┐
│                     训练阶段 (已知 12 类)                          │
│                                                                   │
│  RT×m/z 矩阵 → [预处理] → [Backbone + 双轴注意力] → 嵌入 z      │
│                                                                   │
│  损失函数:                                                        │
│    L_total = L_supcon + λ₁·L_adv + λ₂·L_proto + λ₃·L_recon     │
│                                                                   │
│    L_supcon: 监督对比损失 (同产品拉近、异产品推远)                  │
│    L_adv:   批次对抗损失 (梯度反转, 消除批次信息)                  │
│    L_proto: 原型距离损失 (样本到类原型距离最小化)                  │
│    L_recon: 重建正则损失 (可选, 防止信息丢失)                      │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                     注册阶段 (支持新产品)                          │
│                                                                   │
│  新产品 k 的 N 个参考样本 → [冻结的 Backbone] → {z₁,...,zₙ}     │
│                                                                   │
│  原型: p_k = mean(z₁,...,zₙ)                                     │
│  半径: r_k = percentile(‖zᵢ - p_k‖, 95)                         │
│                                                                   │
│  ← 不涉及任何梯度更新，仅前向推理                                 │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                     推理阶段                                      │
│                                                                   │
│  待测样本 x → [预处理] → [Backbone] → 嵌入 z_q                   │
│                                                                   │
│  输出 1 - 产品类型: argmin_k ‖z_q - p_k‖                         │
│  输出 2 - 一致性分数: S = exp(-‖z_q - p_k*‖ / r_k*)             │
│  输出 3 - 拒识判断: 若 min_k ‖z_q - p_k‖ > θ → 未知产品         │
│  输出 4 - 关键峰: Grad-CAM(z_q → p_k*) → RT×m/z 热力图          │
└─────────────────────────────────────────────────────────────────┘
```

### 模型架构细节

```
输入: X ∈ ℝ^{1 × H × W}  (单通道 RT × m/z 矩阵, 如 256 × 512)
  │
  ├── Stem: Conv2d(1→32, 7×7, s=2) → BN → ReLU → MaxPool
  │
  ├── ResBlock_1: [32→64, ×2]       ← 捕获小范围峰形
  ├── AxialAttn_1: RT-attn + m/z-attn  ← 第一次结构化注意力
  │
  ├── ResBlock_2: [64→128, ×2]      ← 中尺度色谱模式
  ├── AxialAttn_2: RT-attn + m/z-attn
  │
  ├── ResBlock_3: [128→256, ×2]     ← 全局指纹
  ├── AxialAttn_3: RT-attn + m/z-attn
  │
  ├── GlobalAvgPool → z ∈ ℝ^256     ← 最终嵌入
  │
  ├── 产品头: z → FC → softmax (训练辅助, 推理可不用)
  ├── 批次头: GRL(z) → FC → softmax (对抗, 仅训练用)
  └── 重建头: z → Deconv → X̂ (可选正则)
```

## 三、实验设计

### 3.1 数据划分策略

| 设置 | 说明 | 目的 |
|------|------|------|
| **闭集划分** | 12 类全部参与训练，按批次划分 train/val/test | 评估跨批次泛化 |
| **开集划分** | 训练用 9 类，验证用 1 类，测试用 2 类 | 评估新产品零训练识别能力 |
| **少样本划分** | 12 类训练，但测试时模拟 N-shot（N=1,3,5,10）注册 | 评估参考样本数量的影响 |

关键: **按批次划分**（同一批次数据不同时出现在 train 和 test），而非随机划分，否则会高估性能。

### 3.2 对比方法

| 类别 | 方法 | 说明 |
|------|------|------|
| **传统化学计量学** | PCA + Mahalanobis 距离 | 工业常用基线 |
| | PLS-DA | 烟草质控经典方法 |
| | SIMCA | 基于 PCA 的软独立分类 |
| **传统 ML** | SVM (RBF kernel) | 在 PCA 降维后的特征上 |
| | Random Forest | |
| **深度学习 Baseline** | ResNet-18 + softmax | 标准闭集 CNN 分类 |
| | ResNet-18 + 对比学习 | 有度量学习但无双轴注意力/批次对抗 |
| **你的方法变体** | 消融 A: 无双轴注意力 | 验证注意力贡献 |
| | 消融 B: 无批次对抗 | 验证批次校正贡献 |
| | 消融 C: 用 softmax 替代原型 | 验证度量学习 vs 闭集分类 |
| | **完整方法** | 全部模块 |

### 3.3 评估指标

#### (1) 产品识别性能

| 指标 | 公式/说明 | 为什么需要 |
|------|-----------|------------|
| **Overall Accuracy** | 正确数 / 总数 | 基本指标 |
| **Macro F1-Score** | 各类 F1 的宏平均 | 类别不平衡时公平评估 |
| **Per-Class Precision/Recall** | 每类单独报告 | 识别薄弱类别 |
| **Confusion Matrix** | 12×12 混淆矩阵 | 分析哪些产品容易混淆 |

#### (2) 开放集与少样本能力（核心差异化指标）

| 指标 | 说明 | 为什么需要 |
|------|------|------------|
| **N-shot Accuracy** | N=1,3,5,10 参考样本下的分类准确率 | 直接衡量"无需重训练"的能力 |
| **Open-Set AUROC** | 已知类 vs 未知类的分离度 | 衡量拒识未知产品的能力 |
| **Open-Set F1 @ 固定 FPR** | 在 5% 假阳率下的 F1 | 实际部署的决策阈值 |
| **新类注册时间** | 从提供参考样本到可用的时间 | 实际效率指标 |

#### (3) 一致性评分质量

| 指标 | 说明 | 为什么需要 |
|------|------|------------|
| **与人工评分的 Spearman 相关系数** | 模型一致性分数 vs 专家打分 | 验证分数的化学合理性 |
| **一致/不一致的 AUROC** | 用分数区分合格/不合格 | 评估分数的判别力 |
| **分数分布可视化** | 合格样本 vs 不合格样本的分数直方图 | 直观展示 |
| **EER (等错误率)** | FAR = FRR 的点 | 阈值选择的参考 |

#### (4) 批次鲁棒性（核心差异化指标）

| 指标 | 说明 | 为什么需要 |
|------|------|------------|
| **跨批次 Accuracy Drop** | 同批次测试 vs 跨批次测试的精度差 | 越小说明批次鲁棒性越好 |
| **t-SNE / UMAP 嵌入可视化** | 着色：按产品类型 vs 按批次 | 直观展示批次效应消除效果 |
| **批次可预测性 (Batch Predictability)** | 用嵌入 z 预测批次标签的准确率 | 越低说明批次信息被消除得越干净 |
| **Silhouette Score (按产品/按批次)** | 聚类质量 | 按产品应高、按批次应低 |

#### (5) 可解释性

| 指标 | 说明 | 为什么需要 |
|------|------|------------|
| **Grad-CAM 高亮区域与已知标志化合物的重叠率** | 定量评估 | 验证模型关注的区域化学上合理 |
| **专家评审一致率** | 请化学专家判断 Grad-CAM 是否有意义 | 主观验证 |
| **案例展示** | 选 3-5 个典型样本展示完整输出 | 论文可读性 |
